About

This is a simple web application that has pulled together several tools. The recommendation engine and web application are written in Python. The former was designed and built using 'content-based filtering' technique, the MovieLens dataset, and loads data from stored CSV's. The latter was built using a Flask framework, and a Bootstrap template. Alterations were made to the made to the HTML, CSS, and JavaScript of this Bootstrap template.

Once built, the environment packages were frozen into a requirements.txt file. A .gitignore file was created to describe the irrelevant files. A Procfile was created to hold the functions to be run upon loading onto Heroku, the Platform as a Service. The code was then commited both GitHub and Heroku, and deployed on Heroku. Lastly, a custom domain was added to the Heroku hosted web app by altering DNS details.



How the statistics works…

Content-based Filtering utilises the features of the films, rather than the features of the customer, to make its recommendations. Based upon the movie they have just watched, the engine recommends the next most similar movie. This is different from Collaborative Filtering, which groups customers based upon the similarities between customers. Such an engine would recommend a movie to a customer based upon what similar customers have watched.

Content-based Filtering uses the combination of Term Frequency (TF) and Inverse Document Frequency (IDF) to determine the relative importance of the movies a customer has watched. To simplify, we use the TF-IDF method to calculate a Vector Space Model, which is a measure of similarity between movies. The number recommended movies is then displayed based upon this measure of similarity and the customer’s input.
